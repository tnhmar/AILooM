## Objective

Implement `ChromaVectorIndex` — the local ChromaDB adapter for `VectorIndexPort`.
Collections are scoped per `(tenant_id, embedding_model_id)` following ADR-007.
This is the primary semantic search backend for local mode.

## Perplexity Prompt — paste this verbatim

You are implementing the ChromaDB vector index adapter for `memory-layer`.
File: `src/memory_layer/storage/vector/local_vector.py`

The `VectorIndexPort` protocol defines:
```python
async def upsert(self, doc: VectorDocument) -> None
async def search(self, query_embedding: List[float], tenant_id: TenantId,
                 k: int, filters: dict) -> List[VectorSearchResult]
async def delete(self, memory_id: MemoryId, tenant_id: TenantId) -> None
```

Use `chromadb` sync client wrapped in `asyncio.to_thread` for async interface.

Collection naming convention (ADR-007):
  `memory_vectors_{tenant_id_short}_{model_id_slug}`
  where tenant_id_short = first 8 chars of tenant_id (lowercased, alphanumeric only)
  and model_id_slug = model_id with non-alphanumeric chars replaced by underscore.

Store per document:
- id = memory_id
- embedding = float vector
- document = content
- metadata = flattened dict: tenant_id, sector, lifecycle_state, embedding_model_id, embedding_dimensions, plus extra metadata keys

Always add {"tenant_id": tenant_id} to filters in search to prevent cross-tenant leakage.
Raise `StorageError` on ChromaDB errors.
Accept optional `chroma_settings` param on __init__ for testability.
