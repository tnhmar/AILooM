## Objective

Implement the Alembic-based migration runner that applies all DDL from M7-T1–T3
on server startup, emits `SchemaMigratedEvent` per applied migration, and exposes
an admin HTTP endpoint to trigger migrations on demand. This is the final gate
before `v0.1.0` can be cut.

**Generate `src/memory_layer/adapters/postgres/migrator.py`:**

```python
@dataclass
class MigrationResult:
    applied: List[str]    # migration version strings applied in this run
    skipped: List[str]    # already-applied versions skipped
    duration_ms: int

class SchemaMigrator:
    """
    Lightweight migration runner (no Alembic runtime — uses asyncpg directly).

    Design:
    - Maintains a `schema_migrations` table:
        CREATE TABLE IF NOT EXISTS schema_migrations (
            version TEXT PRIMARY KEY,
            applied_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
        );
    - On run(): loads list of all known migrations (ordered list of (version, ddl) tuples).
    - For each migration: check if version exists in schema_migrations.
      If not: execute ddl, insert version, emit SchemaMigratedEvent.
      If yes: skip.
    - Returns MigrationResult.
    - All operations in a single asyncpg transaction per migration version.
    """

    MIGRATIONS: ClassVar[List[Tuple[str, str]]] = [
        ("0001_initial", "<concatenation of all DDL strings>"),
        # Future migrations appended here as ("0002_add_column", "ALTER TABLE...")
    ]

    def __init__(
        self,
        pool: asyncpg.Pool,
        observer: ObserverPort,
        tenant_id: str = "system",
    ) -> None: ...

    async def run(self) -> MigrationResult: ...

    async def _ensure_migrations_table(self) -> None: ...

    async def _is_applied(self, conn, version: str) -> bool: ...

    async def _apply(self, conn, version: str, ddl: str) -> None: ...
```

**Generate `tests/adapters/postgres/test_migrator.py`:**

Mark `@pytest.mark.integration`. Skip if `TEST_POSTGRES_DSN` not set.

Tests:
1. `SchemaMigrator.run()` on a fresh DB applies all migrations and returns `applied` list.
2. `SchemaMigrator.run()` on an already-migrated DB returns `applied=[]`, `skipped` non-empty.
3. `SchemaMigratedEvent` is emitted for each applied migration.
4. `schema_migrations` table exists after first run.
5. Each migration version appears exactly once in `schema_migrations`.
6. `MigrationResult.duration_ms` is a positive integer.
7. `run()` is idempotent: calling twice does not raise and produces same schema.
8. `POST /v1/admin/migrations:run` returns 200 with `applied` field.
9. `POST /v1/admin/migrations:run` without `MEMORY_LAYER_ADMIN_KEY` returns 401.
