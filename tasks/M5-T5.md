## Objective

Implement tenant isolation middleware and guard utilities that ensure every inbound
request is resolved to exactly one tenant and that cross-tenant access attempts are
blocked consistently before reaching engine logic. This closes the loop on multi-tenant
correctness at the API boundary.

**Generate `src/memory_layer/api/middleware.py`:**

Implement a lightweight middleware class/function that:
- Resolves tenant via `resolve_tenant_id` for all `/v1/` non-admin routes.
- Stores tenant ID on `request.state.tenant_id`.
- Skips `/healthz`.
- Skips admin routes under `/v1/admin/` because tenant comes from path there.
- If tenant resolution fails, lets the registered exception handler produce the error.

Also expose helper dependency:
```python
def get_request_tenant_id(request: Request) -> str: ...
```
that reads `request.state.tenant_id`.

Update `api/app.py` to install this middleware and to use `get_request_tenant_id`
instead of reading the header manually.
