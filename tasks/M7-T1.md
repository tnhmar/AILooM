## Objective

Implement `PostgresMemoryRecordRepository` and `PostgresFactRepository` —
concrete implementations of `MemoryRecordRepositoryPort` and `FactRepositoryPort`
backed by asyncpg + SQLAlchemy Core (no ORM). Every query must be scoped to
`tenant_id`. Optimistic concurrency on lifecycle transitions.

**Generate `src/memory_layer/adapters/postgres/record_repo.py`:**

Table schema (emit as module-level constant `MEMORY_RECORDS_DDL: str`):

```sql
CREATE TABLE IF NOT EXISTS memory_records (
    memory_id       TEXT        NOT NULL,
    tenant_id       TEXT        NOT NULL,
    raw_payload     TEXT        NOT NULL,
    payload_type    TEXT        NOT NULL,
    sector          TEXT        NOT NULL,
    lifecycle_state TEXT        NOT NULL,
    pipeline_status TEXT        NOT NULL,
    scope_agent_id  TEXT,
    scope_user_id   TEXT,
    scope_session_id TEXT,
    idempotency_key TEXT,
    recorded_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (memory_id, tenant_id)
);
CREATE INDEX IF NOT EXISTS idx_memory_records_tenant_lifecycle
    ON memory_records (tenant_id, lifecycle_state);
CREATE UNIQUE INDEX IF NOT EXISTS idx_memory_records_idempotency
    ON memory_records (tenant_id, idempotency_key)
    WHERE idempotency_key IS NOT NULL;
```

Implement `PostgresMemoryRecordRepository` satisfying `MemoryRecordRepositoryPort`:

- `save(record)`: INSERT ... ON CONFLICT DO NOTHING.
- `get_by_id(memory_id, tenant_id)`: SELECT WHERE memory_id=$1 AND tenant_id=$2.
- `update_lifecycle(memory_id, tenant_id, state, actor)`:
  UPDATE ... WHERE memory_id=$1 AND tenant_id=$2 AND lifecycle_state != $3
  (optimistic: skip silently if already in target state).
- `update_pipeline_status(memory_id, tenant_id, status)`: UPDATE pipeline_status.
- `list_by_scope(scope, lifecycle_states, limit)`: SELECT with tenant+scope filters.
- `get_by_idempotency_key(key, tenant_id)`: SELECT WHERE idempotency_key=$1 AND tenant_id=$2.

All methods must include `tenant_id` in every WHERE clause — no exceptions.

```sql
CREATE TABLE IF NOT EXISTS facts (
    fact_id          TEXT        NOT NULL,
    tenant_id        TEXT        NOT NULL,
    memory_record_id TEXT        NOT NULL,
    subject_entity_id TEXT       NOT NULL,
    predicate        TEXT        NOT NULL,
    predicate_group  TEXT        NOT NULL,

    sector           TEXT        NOT NULL,
    lifecycle_state  TEXT        NOT NULL,

    effective_to     TIMESTAMPTZ,
    superseded_by    TEXT,
    PRIMARY KEY (fact_id, tenant_id)
);
CREATE INDEX IF NOT EXISTS idx_facts_entity_predicate
    ON facts (tenant_id, subject_entity_id, predicate_group)
    WHERE effective_to IS NULL;
```

Implement `PostgresFactRepository` satisfying `FactRepositoryPort`:

- `save(fact)`: INSERT.
- `get_by_id(fact_id, tenant_id)`: SELECT.
- `close_fact(fact_id, tenant_id, effective_to, new_fact_id)`:
  UPDATE SET effective_to=$3, superseded_by=$4 WHERE fact_id=$1 AND tenant_id=$2.
- `get_active_facts_by_entity_predicate(entity_id, predicate_group, tenant_id)`:
  SELECT WHERE effective_to IS NULL AND subject_entity_id=$1
  AND predicate_group=$2 AND tenant_id=$3.
- `list_by_memory_record(memory_record_id, tenant_id)`: SELECT.

5. `update_pipeline_status` changes status.
6. `list_by_scope` returns only matching tenant+scope records.
7. `list_by_scope` does not return records from a different tenant.
8. `get_by_idempotency_key` returns matching record.
9. `get_by_idempotency_key` returns None when key not found.
10. `get_by_id` returns None for unknown memory_id.

Fact repo tests:
1. `save` then `get_by_id` returns the saved fact.
2. `close_fact` sets `effective_to` and `superseded_by`.
3. `get_active_facts_by_entity_predicate` returns only open facts.
4. Closed fact is NOT returned by `get_active_facts_by_entity_predicate`.
5. `list_by_memory_record` returns all facts for a record.
6. Cross-tenant isolation: fact from tenant A not visible to tenant B.
