## Objective

Implement `LifecycleScheduler` — a lightweight asyncio-based background job runner
that triggers `DecayService` and `ConsolidationService` on configurable intervals
per tenant. Designed to run inside the same process as the engine (no external queue
necessary for the local adapter tier). Must support start/stop lifecycle, per-tenant
isolation, and graceful shutdown.

```python
@dataclass
class ScheduleConfig:
    decay_interval_seconds: int = 3600        # 1 hour default
    consolidation_interval_seconds: int = 1800  # 30 min default
    enabled: bool = True

class LifecycleScheduler:
    """
    Asyncio-based background scheduler for decay and consolidation sweeps.

    Design:
    - Maintains one asyncio.Task per (tenant_id, job_type) pair.
    - Jobs loop: sleep(interval) → run service → repeat.
    - On exception inside a job: log error, do NOT cancel the loop.
    - start(tenant_id, config): creates tasks for decay + consolidation
      (only if ConsolidationTrigger.SCHEDULE is configured — check policy_repo).
    - stop(tenant_id): cancels tasks for that tenant, awaits cancellation.
    - stop_all(): cancels and awaits all running tasks.
    - is_running(tenant_id) -> bool: returns True if tasks exist for tenant.

    Thread-safety: designed for single-event-loop use only.
    Do not add threading primitives.
    """

    def __init__(
        self,
        decay_service: DecayService,
        consolidation_service: ConsolidationService,
        policy_repo: TenantPolicyRepositoryPort,
    ) -> None: ...

    async def start(
        self, tenant_id: TenantId, config: Optional[ScheduleConfig] = None
    ) -> None: ...

    async def stop(self, tenant_id: TenantId) -> None: ...

    async def stop_all(self) -> None: ...

    def is_running(self, tenant_id: TenantId) -> bool: ...

    async def _decay_loop(
        self, tenant_id: TenantId, interval: int
    ) -> None: ...

    async def _consolidation_loop(
        self, tenant_id: TenantId, interval: int
    ) -> None: ...
```
