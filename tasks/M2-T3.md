## Objective

Implement `SqliteFactRepository` — the SQLite adapter for `FactRepositoryPort`.
The key operation is `close_fact`: setting `effective_to` on an old fact and recording
the `supersedes` pointer on the new one. This is the append-only temporal model in action.

`close_fact` must run inside a single transaction:
1. Verify fact exists and belongs to tenant_id.
2. SET effective_to on old fact.
3. SET supersedes = fact_id on new fact row.
Roll back entirely if either update fails.
`get_active_facts_by_entity_predicate` returns only rows where lifecycle_state='ACTIVE' AND effective_to IS NULL.
