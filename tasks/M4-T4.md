## Objective

Implement `ExplainRecallService` — the concrete `ExplainRecallUseCase` that retrieves
a `MemoryTrace` for a given `trace_id`, reconstructing the full retrieval story:
which memories were considered, which were returned, what scores and signals drove
the decision, and the `QueryPlan` used. This makes the memory layer fully auditable
and debuggable by agents and operators.

```python
class ExplainRecallService:
    """
    Implements ExplainRecallUseCase.

    Flow:
    1. Call trace_repo.get_by_trace_id(trace_id, tenant_id).
    2. If not found: raise MemoryNotFoundError(f"Trace {trace_id} not found").
    3. For each TraceStep in the trace:
       a. Attempt to load the associated MemoryRecord via record_repo.get_by_id.
       b. If the record no longer exists (deleted), mark step.record_available=False.
       c. Attach record metadata to step if available.
    4. Return the fully populated MemoryTrace.
    """

    def __init__(
        self,
        trace_repo: TraceRepositoryPort,
        record_repo: MemoryRecordRepositoryPort,
        audit_log: AuditLogPort,
    ) -> None: ...

    async def execute(
        self, trace_id: TraceId, tenant_id: TenantId
    ) -> MemoryTrace: ...
```
