## Objective

Implement an MCP server exposing memory operations as tools for agentic IDEs and
assistants. This creates a native tool interface for Claude Code / Cursor / Windsurf-style
workflows and makes the memory layer directly consumable via MCP.

---

## Perplexity Prompt — paste this verbatim

You are implementing an MCP server for `memory-layer`.
File: `src/memory_layer/mcp/server.py`.

The project should expose tools backed by `MemoryLayerClient`.
Assume the MCP runtime library is available as `from mcp.server.fastmcp import FastMCP`.
Use stdlib + this runtime only.

---

**Generate `src/memory_layer/mcp/server.py`:**

Implement:

```python
mcp = FastMCP("memory-layer")

# tool functions
async def write_memory(...)
async def search_memory(...)
async def recall_memory(...)
async def explain_trace(...)
async def end_session(...)
async def get_memory(...)
async def delete_memory(...)
```

Rules:
- Register each tool with the MCP server.
- Each tool should call `MemoryLayerClient`.
- Tenant ID must be supplied explicitly as a tool argument.
- Tool inputs should be simple JSON-serializable primitives / dicts.
- Return JSON-serializable dicts only.
- Provide concise docstrings so MCP clients can render tool help.
- Add `build_client(tenant_id: str) -> MemoryLayerClient` helper as a seam for tests.
- Do not hardwire deployment config; read base URL from env var `MEMORY_LAYER_BASE_URL`
  with default `http://localhost:8000`.
```

---

**Generate `tests/mcp/test_server.py`:**

Use `pytest-asyncio`. Monkeypatch `build_client` to return a fake async client.

Tests:
1. MCP module defines `mcp` server object.
2. `write_memory()` returns JSON-serializable dict with `memory_id`.
3. `search_memory()` returns JSON-serializable result payload.
4. `recall_memory()` returns JSON-serializable result payload.
5. `explain_trace()` returns JSON-serializable trace payload.
6. `get_memory()` delegates to client and returns dict.
7. `delete_memory()` returns success payload.
8. `end_session()` returns success payload.
9. `build_client()` reads `MEMORY_LAYER_BASE_URL` env var.
10. All tool functions require explicit `tenant_id` parameter.
