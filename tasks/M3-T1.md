## Objective

Implement `WriteMemoryService` — the concrete use case that satisfies
`WriteMemoryUseCase`. It orchestrates: idempotency check → durable raw write →
audit entry → event emission → async enrichment trigger.
This is the hot write path: synchronous portion must be fast; enrichment is always async
unless `wait_for_enrichment=True`.

**Generate `src/memory_layer/engine/ingestion.py`:**

```python
class WriteMemoryService:
    """
    Implements WriteMemoryUseCase.

    Write flow:
    1. Check idempotency key — if hit, return cached WriteResult with idempotent=True.
    2. Infer sector if not provided (default: EPISODIC for CONVERSATION_TURN, SEMANTIC otherwise).
    3. Construct MemoryRecord with lifecycle_state=ACTIVE, pipeline_status=PENDING.
    4. Save to MemoryRecordRepositoryPort (durable write).
    5. Append AuditEntry (operation=WRITE, outcome=SUCCESS).
    6. Emit MemoryWrittenEvent via ObserverPort.
    7. If wait_for_enrichment=True: await _enrich(record) directly.
       Else: schedule _enrich(record) as a background asyncio task.
    8. Return WriteResult(memory_id, scope, pipeline_status=PENDING, accepted_at=now).

    Enrichment (_enrich):
    - Call ExtractionPort.extract(record).
    - On success: update pipeline_status to ENRICHED via repo.
                  Emit MemoryEnrichedEvent.
    - On failure: update pipeline_status to PARTIAL_ENRICHMENT_FAILED.
                  Emit EnrichmentFailedEvent.
                  Log warning — do NOT propagate exception.
    """

    def __init__(
        self,
        record_repo: MemoryRecordRepositoryPort,
        audit_log: AuditLogPort,
        observer: ObserverPort,
        extraction: Optional[ExtractionPort] = None,
    ) -> None: ...

    async def execute(self, request: WriteRequest) -> WriteResult: ...

    async def _enrich(self, record: MemoryRecord) -> None: ...
```

Rules:
- `WriteMemoryService` must satisfy `isinstance(svc, WriteMemoryUseCase)`.
- Sector inference: `CONVERSATION_TURN` → `EPISODIC`; `DOCUMENT` → `SEMANTIC`;
  `TOOL_OUTPUT` → `PROCEDURAL`; `EVENT` → `EPISODIC`; `STRUCTURED` → `SEMANTIC`.
  If `request.sector` is provided explicitly, use that value always.
- If `extraction` is `None` and `request.extract=True`, set `pipeline_status=ENRICHMENT_SKIPPED`.
- Idempotency check must call `record_repo.get_by_idempotency_key`.
- All async background tasks must be created with `asyncio.create_task`.
- Log enrichment exceptions at `WARNING` level with `exc_info=True`.
