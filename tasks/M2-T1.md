## Objective

Create the canonical SQLite schema (V1 DDL) and a lightweight migration runner
that tracks schema version in a `schema_version` table.
This is the foundation every other local storage adapter builds on.

---

**File 2 ā€” migration_runner.py**

Implement (raw `sqlite3`, no ORM):

```python
def _parse_version(filename: str) -> int:
    """Extract integer version from filename like V1__description.sql."""

def get_current_version(conn: sqlite3.Connection) -> int:
    """Return highest applied migration version, or 0 if schema_version table absent."""

def run_migrations(conn: sqlite3.Connection) -> list[int]:
    """Apply all pending .sql files from migrations/ dir in version order.
    Each migration runs in its own transaction. Returns list of applied versions."""

def ensure_schema(db_path: str) -> sqlite3.Connection:
    """Open connection, set PRAGMA journal_mode=WAL and foreign_keys=ON,
    run all pending migrations, return connection."""
```

Raise `SchemaVersionError` (domain/exceptions.py) if DB is ahead of known migrations.

**Generate `tests/storage/test_migration_runner.py`** with 12 tests:

1. `ensure_schema(":memory:")` succeeds and returns a connection.
2. After `ensure_schema`, `get_current_version` returns `1`.
3. Running `ensure_schema` a second time is idempotent.
4. All expected tables exist: `memory_records`, `facts`, `audit_log`, `tenant_policies`.
5. `idx_mr_tenant_lifecycle` index exists.
6. `idx_facts_tenant_entity` index exists.
7. Unique index on `idempotency_key` enforces uniqueness per tenant.
8. Inserting a row with `lifecycle_state` omitted defaults to `'ACTIVE'`.
9. `get_current_version` returns `0` on a fresh empty DB.
10. `_parse_version("V1__initial_schema.sql")` returns `1`.
11. `_parse_version("V12__add_embeddings.sql")` returns `12`.
12. `_parse_version("bad_name.sql")` raises `ValueError`.
